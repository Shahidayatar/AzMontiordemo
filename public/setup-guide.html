📋 Application Insights Setup Guide

Complete step-by-step guide to configure your Application Insights Dashboard

1

Create Application Insights Resource

Go to Azure Portal:

  1. Navigate to portal.azure.com
  2. Click "Create a resource"
  3. Search for "Application Insights"
  4. Click "Create"
  5. Fill in the details:
    • Subscription: Your Azure subscription
    • Resource Group: Create new or use existing
    • Name: e.g., "my-app-insights"
    • Region: Choose closest to your app
    • Resource Mode: Workspace-based
  6. Click "Review + Create" → "Create"
📸 Screenshot: Azure Portal > Create Application Insights
2

Get Connection String

From your Application Insights resource:

  1. Go to your Application Insights resource
  2. In the left menu, click "Overview"
  3. Find "Connection String" in the Essentials section
  4. Click the copy button next to the connection string
InstrumentationKey=01234567-89ab-cdef-0123-456789abcdef;IngestionEndpoint=https://polandcentral-0.in.applicationinsights.azure.com/;LiveEndpoint=https://polandcentral.livediagnostics.monitor.azure.com/;ApplicationId=01234567-89ab-cdef-0123-456789abcdef

⚠️ Important

Keep this connection string secure! It's used to send telemetry data to Azure.

3

Get Application ID

From your Application Insights resource:

  1. In the left menu, click "API Access"
  2. Copy the "Application ID" at the top
Application ID: 01234567-89ab-cdef-0123-456789abcdef
📸 Screenshot: Application Insights > API Access > Application ID
4

Create API Key

Still in the API Access page:

  1. Click "Create API Key" button
  2. Enter a name: Dashboard-API-Key
  3. Select permissions: ✅ Read telemetry
  4. Click "Generate key"
  5. IMMEDIATELY COPY THE KEY - you won't see it again!
API Key: abc123def456ghi789jkl012mno345pqr678stu

🔒 Critical

Copy the API key immediately! Azure only shows it once. If you lose it, you'll need to create a new one.

5

Configure Environment Variables

Create or update your .env file:

# Application Insights Configuration APPINSIGHTS_CONNECTION_STRING=InstrumentationKey=01234567-89ab-cdef-0123-456789abcdef;IngestionEndpoint=https://polandcentral-0.in.applicationinsights.azure.com/;LiveEndpoint=https://polandcentral.livediagnostics.monitor.azure.com/;ApplicationId=01234567-89ab-cdef-0123-456789abcdef APP_INSIGHTS_APP_ID=01234567-89ab-cdef-0123-456789abcdef APP_INSIGHTS_API_KEY=abc123def456ghi789jkl012mno345pqr678stu # Optional PORT=3000 NODE_ENV=development

Replace the example values with your actual:

  • Connection string from Step 2
  • Application ID from Step 3
  • API Key from Step 4
6

Test Your Configuration

Start the application and verify:

npm start

Look for these success messages:

✅ Application Insights initialized successfully 🚀 Azure Monitor Dashboard STARTED ⚙️ Configuration Status: APPINSIGHTS_CONNECTION_STRING: ✅ SET APP_INSIGHTS_APP_ID: ✅ SET APP_INSIGHTS_API_KEY: ✅ SET

✅ Success Indicators

  • Server starts without errors
  • All configuration shows ✅ SET
  • Dashboard loads at http://localhost:3000
  • Status shows "Connected" (not "Connecting...")
7

Generate Test Data

Create some telemetry to see in your dashboard:

  1. Click "Trigger Error" button 3-5 times
  2. Click "Slow Endpoint" button 3-5 times
  3. Click "Health Check" button 3-5 times
  4. Wait 2-5 minutes for data to appear in Application Insights
  5. Click "Refresh" to update charts

📈 Expected Results

  • Charts show data points (not flat lines)
  • Statistics cards show numbers > 0
  • Status changes to "Connected"
  • No "Using sample data" warnings
8

Troubleshooting

🔧 Common Issues

Charts are empty/flat:

  • Wait 2-5 minutes for Application Insights data processing
  • Generate more test data using the buttons
  • Check Azure Portal → Application Insights → Logs for data

"Using sample data" message:

  • Check API key has "Read telemetry" permission
  • Verify Application ID is correct
  • Ensure API key hasn't expired

Connection errors:

  • Verify internet connection
  • Check connection string format
  • Ensure Application Insights resource exists

🎉 You're All Set!

Your Application Insights Dashboard is now ready to monitor your applications.
